# Requirements: Gateway Runtime Ops

> **目标**：建立 API/MCP 网关的发布、灰度、回滚和运维标准。  
> **优先级**：P0  
> **预估工作量**：3-5 天

---

## 1. 用户故事

- 作为 SRE，我希望协议发布可灰度可回滚，这样能控制故障爆炸半径。  
- 作为平台 owner，我希望网关健康状态可观测，这样能快速定位故障。

---

## 2. 功能需求

### FR-1 发布门控
- [ ] 定义 canary -> 扩量 -> 全量流程
- [ ] 定义自动回滚触发条件（错误率/时延/SLO）

### FR-2 运维手册
- [ ] 定义网关启动、探针、故障分级、回滚指令
- [ ] 定义值班排障路径与升级路径

### FR-3 观测指标
- [ ] 定义协议层核心指标（QPS、P95、错误率、超时率）
- [ ] 定义 SLO 与错误预算

---

## 3. Definition of Done

- [ ] 运行手册完成并通过一次演练
- [ ] SLO 仪表盘可用
- [ ] 回滚流程可在演练中成功执行
