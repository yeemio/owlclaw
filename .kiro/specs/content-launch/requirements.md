# 内容营销启动 — 需求文档

> **Spec**: content-launch
> **阶段**: Phase 8.3
> **决策来源**: `docs/DUAL_MODE_ARCHITECTURE_DECISION.md` §6.7 内容营销 + §7 咨询实施
> **前置**: mionyee-governance-overlay + mionyee-hatchet-migration（需要真实数据）

---

## 1. 背景

内容是一人公司最重要的杠杆。参考 Jason Liu（jxnl）的经验：一篇技术文章在 HN 获得 30,000 次浏览，直接成为咨询业务的最大获客来源。

本 spec 覆盖三个交付物：第一篇技术文章、Mionyee 案例材料、咨询方案模板。

## 2. User Stories

### US-1：第一篇技术文章
**作为** OwlClaw 项目维护者，**我希望** 发布一篇解决具体痛点的技术文章，**以便** 获取第一批开发者关注。

**验收标准**：
- [ ] 文章解决一个具体痛点（非产品介绍）
- [ ] 文章结构：问题 → 我试过的方案 → 我的解决方案 → 结果 → 你也可以试试
- [ ] 文章附带可运行代码（pip install owlclaw，3 步上手）
- [ ] 发布到至少 2 个渠道（Reddit/HN/掘金/V2EX）

### US-2：Mionyee 案例材料
**作为** OwlClaw 项目维护者，**我希望** 有一份基于 Mionyee 真实数据的案例材料，**以便** 用于咨询获客和内容营销。

**验收标准**：
- [ ] 案例包含真实数据：治理后成本降低 X%、调度稳定性提升 Y%
- [ ] 案例格式：问题 → 方案 → 实施 → 结果（数据对比）
- [ ] 可用于两个场景：技术文章素材 + 咨询方案附件

### US-3：咨询方案模板
**作为** OwlClaw 项目维护者，**我希望** 有一套标准化的"AI 智能化转型"咨询方案模板，**以便** 快速响应咨询需求。

**验收标准**：
- [ ] 模板覆盖完整流程：调研 → 方案 → 实施 → 交付 → 维护
- [ ] 模板可参数化（客户名称、系统类型、场景类型可替换）
- [ ] 包含定价参考（培训 / 项目实施 / 月维护）

## 3. 非功能需求

- **时效性**：第一篇文章在 Phase 8.2 完成后 2 周内发布
- **真实性**：所有数据必须来自 Mionyee 真实运行数据，禁止编造
- **可复现**：文章中的代码示例必须可运行

## 4. Definition of Done

- [ ] 第一篇技术文章已发布到至少 2 个渠道
- [ ] Mionyee 案例材料已完成（含真实数据）
- [ ] 咨询方案模板已完成（可参数化）
- [ ] 文章中的代码示例可运行
