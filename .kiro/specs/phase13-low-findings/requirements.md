## 3. 功能需求

### 3.1 低优先级安全与健壮性收口

#### FR-L1：Langfuse secret 配置暴露治理（Finding #11）

**需求**: 配置对象、日志与 API 输出不得暴露 Langfuse 密钥原文。

**验收标准**:
- [ ] 配置导出路径使用掩码值（如 `***`）
- [ ] 单元测试覆盖敏感字段掩码逻辑

#### FR-L2：`_is_select_query` 启发式误判治理（Finding #12）

**需求**: SQL 只读判定应减少启发式误判，避免将非只读语句错误放行。

**验收标准**:
- [ ] 对多语句/注释绕过/大小写混淆场景判定稳定
- [ ] 单元测试覆盖典型绕过样例

#### FR-L3：Shadow mode 查询泄露防护（Finding #13）

**需求**: shadow 执行链路不得泄露敏感查询结果到主决策上下文。

**验收标准**:
- [x] shadow 输出遵循最小可见字段
- [x] 集成测试验证 shadow 不泄露敏感数据

#### FR-L4：Heartbeat DB I/O 优化（Finding #14）

**需求**: heartbeat 数据库查询应可控，避免高频 I/O 影响主流程。

**验收标准**:
- [x] 增加最小轮询间隔或轻量缓存机制
- [x] 单元/集成测试验证行为与阈值

## 8. 风险与缓解

### 8.1 风险：SQL 判定规则收紧导致误拦截
**影响**: 可能影响历史合法查询。

**缓解**:

**影响**: 可能增加部分触发延迟。

**缓解**:
- 通过配置控制间隔，并增加边界测试。
